What is Virtualization

Overview

At first glance, virtualization is certainly one of those terms that give beginner developers a tough time to wrap their heads around, and can be even more challenging to get started implementing. However, virtualization is easy to comprehend with a little time, and can become a powerful tool in a developers toolbox. This webpage will go into broad detail about what exactly virtualization is.

What is Virtualization Exactly?

It gave developers in the past the ability to run multiple operating systems in different virtual machines running simultaneously on the same host. This also allowed developers to do the same work without the need of extra hardware resources.

What are some pros and cons of using Virtualization?

To start, the fact that it allows developers to run multiple operating systems at the same time is a plus, as it allows them to perform different actions across different systems. This also benefits them in case of failure, as it is easy to recover and maintain the virtual machines. This all in turn keeps costs down, due to the decreasing need and usage for infrastructure With all the positives come the negatives, as well. Running multiple virtual machines at the same time on the host operating system can lead to performance degradation. The guest operating system (virtual machine) runs on top of the host operating system, and has its own kernel and dependencies. What this means is that the guest operating system will take up a large portion of the system resources, including the Hard Disk, Processor and RAM. In turn, the host operating system will have less resources to perform at optimal performance. The overall efficiency of the hypervisors are not as good as the host operating, as it does not use the same amount of resources as the host operating system normally uses. The virtual machines take a long time to boot-up, in comparison to the normal boot-up time of the host operating system
